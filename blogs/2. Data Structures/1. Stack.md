<!-- # Stack ( LIFO ) -->


A linear data structure that follows the principle of **Last In First Out (LIFO)**. 

The element inserted at last will be removed first that is all. 
Even though it's working is pretty simple, it is widely used in your day-to-day software. From OS to the a basic mobile application this is very useful. features like going to previous page in your browser or a mobile app, checking whether the syntax of your code is correct or not, the **“CTRL + Z”** that you always use are all enabled by a stack. 

## The last-in-first-out principle.

whoever came last, will leave first. 

the last element you added (pushed) in your stack will be thrown (popped) out first. 

## Basic Operations of Stack

There are some basic operations that allow us to perform different actions on a stack.

- **Push**: Add an element to the top of a stack
- **Pop**: Remove an element from the top of a stack
- **IsEmpty**: Check if the stack is empty
- **IsFull**: Check if the stack is full
- **Peek**: Get the value of the top element without removing it

### How to implement
---
1. A pointer called TOP is used to keep track of the top element in the stack. Both pushing and popping an element is done at the top of the stack.
2. When initializing the stack, we set the value of TOP to -1 ( so that we can check if the stack is empty by comparing TOP == -1.)
3. While pushing an element, 
    - check if the stack is already full
    - increase the value of TOP and 
    - place the new element in the position pointed by TOP. 
4. On popping an element, 
    - check if the stack is already empty
    - we return the element pointed by TOP 
    - decrement the value of TOP.

    ```java
    class Stack {
        int top = -1;
        int arr[] = new int[100];
    		
    		boolean isEmpty() { 
    		    return top == -1;
        }
        boolean isFull(){
    		    return top == arr.length - 1;
        }
        
        void push(int x) {
            if (this.isFull()) { 
    	        System.out.println("Stack is full"); 
    	        return;
    				}
    				top++;
            arr[top] = x;
        }
        int pop() {
            if (this.isEmpty()) { 
    		        System.out.println("Stack is empty"); 
    		        return -1; 
    		    }
            return arr[top--];
        }
        int peek() { 
    		    return this.isEmpty() ? -1 : arr[top]; 
        }
    }
    ```
    

### Use

- To reverse a word
- solve expressions present in the prefix or postfix form.
- Reversing a string/word
- Evaluating prefix/postfix expressions
- Infix → Postfix/Prefix conversion
- Balanced parentheses / bracket matching
- Function call management (call stack, recursion)
- Undo/Redo operations in editors
- Backtracking algorithms (maze, N-Queens, DFS)
- Browser back/forward history

### Types

- **Array-based Stack** – fixed size
- **Linked List-based Stack** – dynamic size
- **Dynamic Stack (resizable array)** – ArrayList



### Stack vs. Queue
---
| Feature | Stack | Queue |
| :--- | :--- | :--- |
| **Core Principle** | **LIFO** (Last In, First Out) | **FIFO** (First In, First Out) |
| **Access Point** | Only from the **Top** | Entry at **Rear**, Exit at **Front** |
| **Analogy** | A stack of dinner plates | A line at a grocery store checkout |
| **Primary Methods** | `push()` and `pop()` | `enqueue()` and `dequeue()` |
