<!-- # Queue ( FIFO ) -->
Queue is a linear data structure that follows the principle of **First In First Out (FIFO)**.

First In First Out
---
    - insertion happens at one end (REAR). 
    - deletion at the other end (FRONT).
ie. we add at one end and remove from the other end.
it's like a waiting for buying the new Iphone outside the apple store, or trying to book the tatkal ticket, whoever comes first will be served first. 

Below is the list of the basic operations / functions a queue need to allow.

### Basic Operations

- **Enqueue**: Add element at rear (end). 
- **Dequeue**: Remove element from front
- **Peek**: View front element without removing. (sneeking)
- **isEmpty**: Check if queue is empty.
- **isFull**: Check if queue reached max capacity (array implementation)

### How to implement ( array )

1. Initially FRONT = 0, REAR = -1
4. If the FRONT > REAR, then the queue does not have any element. ie the queue is empty.
    ```java
    public boolean isEmpty() { 
        return front > rear; 
    }
    ```
5. To check if the queue is full, you just need to check if the REAR's value 
    ```java
    boolean isFull(){
        return rear == arr.length -1;
    }

    ```
3. While Enqueue: 
    1. check if queue is full  ( REAR == size-1 ) 
    2. increment the REAR++, 
    3. add the element at REAR 

    ```java
    void enqueue(int x) {
        if (this.isFull()) { 
            System.out.println("Queue is full"); 
            return; 
        }
        arr[++rear] = x;
    }
    ```

4. While Dequeue: 
    1. check is queue is empty ( FRONT > REAR )  
    2. return the element at FRONT 
   3. increment the value of FRONT++. 
   ```java
   int dequeue() {
        if (this.isEmpty() { 
            System.out.println("Queue is empty."); 
            return -1; 
        }
        return arr[front++];
    }
   ```
5. For peek, 
    1. check if queue is empty or not
    2. If queue is empty return -1.
    3. Otherwise return the element at FRONT.
    ```java
    int peek() { 
        return this.isEmpty() ? -1 : arr[front];
    }
    ```

|  wasted space after dequeues → solved by **Circular Queue / linked list** 


Here is the complete quque implementation, try yourself first though. 

```java
class Queue {
        int front = 0, rear = -1;
        int arr[] = new int[100];
    		
        boolean isEmpty() { 
    	    return front > rear; 
        }
        
        boolean isFull(){
    		return rear == arr.length -1;
    	} 
    		
        void enqueue(int x) {
            if (this.isFull()) { 
                System.out.println("Queue is full"); 
                return; 
            }
            arr[++rear] = x;
        }
        
      
        int dequeue() {
            if (this.isEmpty() { 
                System.out.println("Queue is empty."); 
                return -1; 
            }
            return arr[front++];
        }
        int peek() { 
    		return this.isEmpty() ? -1 : arr[front];
        }
    }
```

### Types of Queue

1. **Simple Queue** – (array)
2. **Circular Queue** ( linked list ) 
— last position connects back to first, avoids wasted space 
3. **Priority Queue** (heap)
— elements have priority, 
— highest priority served first 
4. **Deque (Double Ended Queue)**  
— insertion/deletion possible from both ends.
