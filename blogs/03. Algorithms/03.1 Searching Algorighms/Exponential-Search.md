# Exponential Search

Search through unbounded or infinite arrays.

→ indexes : 1, 2, 4, 8, 16, 32, 64….. increases exponentially. 

**Time Complexity**
• **Best-Case Complexity: O(1)**
    ◦ The target is at index 0 or index 1, hitting immediately.
• **Worst-Case & Average-Case Complexity: O(log i)**
    ◦ Where i is the actual index of the target element in the array.
